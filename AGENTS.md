## Spec-Driven Development (SDD) — MANDATORY

This project uses spec-driven development. All agents must follow these rules.

### Spec Layer Structure

```
specs/
  entities/   ← Zod schemas — SINGLE SOURCE OF TRUTH for all types
  features/   ← Behavior scenarios — source of truth for feature logic
  fixtures/   ← Deterministic typed mock data used in tests and effects
```

### Mandatory Rules

1. **Schema first** — edit `specs/entities/*.schema.ts` before touching code. Never define `interface` or `type` manually.
2. **Fixtures over inline mocks** — all mock data goes in `specs/fixtures/`. Effects import from there.
3. **Scenario before test** — add scenario to `specs/features/*.scenarios.ts` before writing test code.
4. **FSD direction**: `specs → shared → entities → features → widgets → app`. Never reverse.
5. **Canonical types**: `Event.time` (HH:MM string), `Notification.read` (boolean). See schemas.

### Workflow

```
specs/entities/ → specs/fixtures/ → specs/features/ → features/*/model.ts → __tests__/ → app/
```
